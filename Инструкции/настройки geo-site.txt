Как сделать правильно для Украины:
Для вашей задачи (автоматически обновлять списки заблокированных RU-ресурсов) лучше использовать прямые источники баз .dat.
Рекомендую заменить текущие ссылки на эти (удалите пробелы):
В поле «Источник файлов Geo (необязательно)»:
https : // github . com / Loyalsoldier / v2ray - rules - dat / releases / latest / download / geoip . dat
В поле «Источник файлов наборов правил sing-box (необязательно)»:
https : // github . com / Loyalsoldier / v2ray - rules - dat / releases / latest / download / geosite . dat
⚡️ Почему это лучше?
Актуальность: Loyalsoldier обновляет списки каждые 24 часа. Хабр и новые .com домены там точно есть.
Чистота: Вам не нужны сложные китайские или специфические российские шаблоны, которые могут конфликтовать с вашим провайдером в Одессе.
Простота: Вы один раз прописываете правила geosite:category-ru -> proxy и geosite:ua -> direct, и программа сама знает, что делать.
💡 Важное действие:
После того как замените ссылки и нажмете «Подтвердить»:
Обязательно установите «Интервал автоматического обновления Geo» на 24 (сейчас у вас стоит 0, и автообновление выключено).
Нажмите кнопку «Проверить обновление» в главном меню, чтобы скачать новые базы прямо сейчас.
Если при обновлении вы увидите ошибку в логах (красным цветом), значит, программа не может достучаться до GitHub напрямую. В этом случае включите прокси в системном трее и повторите попытку.

 Финальный алгоритм «Оживления»:
Очистите всё лишнее:
Удалите все ссылки из полей «Источник файлов Geo» и «Источник файлов наборов правил...». Сделайте поля пустыми.
Нажмите «Подтвердить».
Пропишите ПРАВИЛЬНЫЕ ссылки (одну за другой):
Вставьте в первое поле (Geo):
https : // github . com / Loyalsoldier / sing - geoip / releases / latest / download / geoip . db
Вставьте во второе поле (наборы правил):
https : // github . com / Loyalsoldier / sing - geosite / releases / latest / download / geosite . db
СРАЗУ нажмите «Подтвердить», не переключая другие вкладки.
Принудительное обновление:
В главном окне (сверху): Проверить обновление -> Обновить geoip и geosite.
Смотрите в лог: Если напишет Success, файлы geoip.db и geosite.db ОБЯЗАНЫ появиться в папке guiConfigs.
🔍 Почему Хабр шел через direct (в Одессе):
В логах, что вы прислали раньше, было outbound/direct. Это значит, что программа не нашла Хабр в своих текущих базах (так как базы были старые или пустые) и пустила его «напрямую». А «напрямую» у вас — таймаут.
Как только вы обновите базы по ссылкам выше:
Зайдите в «Настройки маршрутизации».
В правиле для прокси в поле «Домен» напишите: geosite:category-ru.
Если Хабр всё равно не пойдет: В этой версии ядра Sing-box в v2rayN иногда багует префикс geosite:. Если не сработает, просто замените geosite:category-ru на ru (короткий код для Sing-box).

Почему мы так и не увидели файлы (финальный вердикт):
В v2rayN v7.20 при использовании sing-box разработчики внедрили систему «Remote Rule-sets».
Программа не скачивает файл на диск.
Она скачивает его в оперативную память или временный кэш базы данных.
При каждом запуске она проверяет контрольную сумму по ссылке.
Итог: Автоматика работает «в облаке» самой программы, поэтому физических файлов geo на диске больше нет. Это не баг, это такая (дурацкая) фича новой версии.
Почему Хабр сопротивлялся:
Хабр сейчас сидит за мощными фильтрами и часто меняет IP. Если база внутри v2rayN хоть немного устарела или не так распарсила домен .com, он вылетает в direct. Твое ручное добавление — самый надежный способ, потому что оно имеет высший приоритет над любыми облачными базами.
